# 报价查询系统 · 设计文档 v2

-----

## 一、系统概述

一个在线报价查询系统，销售人员上传 Excel 报价文件后，可按产品型号关键词跨文件搜索报价信息。

- **本地测试**：VS Code Live Server 打开 `index.html` 即可，无需安装 Node.js 或任何后端
- **部署方式**：推送到 GitHub → 开启 GitHub Pages → 自动上线，完全免费
- **跨平台使用**：安卓、苹果、电脑浏览器均可访问同一网址
- **数据存储**：数据保存在各设备浏览器的 IndexedDB 中，关闭浏览器后数据保留
- **跨设备同步**：不支持，各设备独立存储，每台设备首次使用需各自上传 Excel 文件
- **数据丢失风险**：手动清除浏览器缓存 / 使用无痕模式，数据会丢失

-----

## 二、核心功能

### 2.1 搜索检索

- 用户在搜索框输入型号关键词（如 “388”、“CC388A”）
- 系统在**所有已上传的报价文件**中模糊匹配（匹配「型号」列和所有数据列）
- 结果按 **文件 → 分类 → 数据行** 三级折叠面板展示
- 支持按文件筛选（多选框勾选/取消特定文件）

### 2.2 文件管理（管理后台）

- **上传**：点击上传 Excel 文件，解析后写入 IndexedDB，下次打开网站数据仍在
- **更新**：支持用新 Excel 替换已有报价文件（先删旧数据，再写入新数据）
- **删除**：支持删除文件及其所有关联数据行

### 2.3 结果展示

- 文件级可折叠面板（蓝色标题栏）
- 分类级可折叠面板（灰色标题栏，缩进）
- 数据表格（固定表头、斑马纹、横向滚动）
- 搜索匹配单元格高亮（黄色背景）
- 超长文本（>20字）截断后只显示5个字符加省略号，双击弹窗查看完整内容

## 三、Excel 文件格式规范

```
HP系列              ← 分类标题行（单列，不可超50字，不可纯数字）
序号 | 型号   | 适用机型   | 页产量 | 标准版 | 商务版  ← 表头行（多列）
1   | CC388A | HP M1136  | 1500  | 35    | 42     ← 数据行
2   | Q2612A | HP 1020   | 2000  | 28    | 35     ← 数据行
Canon系列           ← 下一个分类...
序号 | 型号   | ...
```

系统自动识别：

- **分类行**：仅首列有文字、其余列为空的行
- **表头行**：分类之后的第一个多列行
- **型号列**：表头中含”型号/规格/品名”等关键词的列（回退：含字母数字且长度≥3的列）

## 四、前端数据流

### 4.1 搜索流程

```
用户输入 "388" → Enter
  ↓
doSearch() [js/search.js]
  ├─ 获取选中的文件ID列表
  ├─ 对每个文件调用 db.search("388", fileId) [js/db.js]
  │   └─ IndexedDB 遍历 quotation_rows，对 model 和 row_data 所有字段模糊匹配
  ├─ mergeResults() 合并同名文件结果
  └─ renderResults() 渲染三级面板
      ├─ result-file-header: 📁 文件名（XX条）
      │   └─ result-category-header: ▶ 分类名（XX行）—— 默认折叠，点击才渲染表格
      │       └─ table.data-table
      │           ├─ thead: 固定表头（浅蓝背景）
      │           └─ tbody: 数据行（斑马纹、高亮匹配格）
      └─ 双击截断单元格 → showCellDetail() 弹出完整内容
```

### 4.2 上传流程

```
用户点击上传区域 或 拖拽 .xlsx 文件进入
  ↓
FileReader 读取文件内容为 ArrayBuffer
  ↓
SheetJS (xlsx.js CDN) 解析 Excel 工作表
  ├─ 逐行扫描，识别分类行（首列有值，其余列为空，且非纯数字，长度≤50）
  ├─ 识别表头行（紧跟分类行之后的第一个多列行）
  └─ 识别型号列（含"型号/规格/品名"关键词；回退：含字母数字且长度≥3）
  ↓
构建两类数据对象
  ├─ fileRecord:
  │   { file_name, file_hash(SHA-256), row_count, cat_count, created_at }
  └─ rows[]:
      { file_id, file_name, category, category_order,
        model, header_data(Array), row_data(Array), row_order }
  ↓
db.saveFile(fileRecord, rows) [js/db.js]
  ├─ 计算 SHA-256，检查 quotation_files 是否已有同 hash（重复文件提示）
  ├─ 用 IDBTransaction 将 fileRecord 写入 quotation_files
  └─ 批量将所有 rows 写入 quotation_rows（同一事务，保证原子性）
  ↓
Toast 通知：「✅ 上传成功：XX 行 / XX 分类」
刷新文件列表面板
```

## 五、客户端存储架构

### 5.1 部署模式

无服务器，纯静态文件：

```
项目文件夹（所有文件在此，无散落文件）
     │
     ↓  git push
  GitHub 仓库
     │
     ↓  Settings → Pages → 选 main 分支
  GitHub Pages（免费，自动上线）
     │
     ↓  访问 https://你的用户名.github.io/仓库名
  安卓 / 苹果 / 电脑 均可使用
```

本地测试：VS Code 安装 Live Server 插件 → 右键 index.html → Open with Live Server

### 5.2 数据模型（IndexedDB）

数据库名：`QuotationDB`　版本：`1`

**对象存储：quotation_files**（报价文件元信息）

|字段        |类型           |说明               |
|----------|-------------|-----------------|
|id        |自增整数（keyPath）|主键               |
|file_name |string       |文件名，如「科思特报价.xlsx」|
|file_hash |string（唯一索引） |SHA-256，用于去重     |
|row_count |number       |数据行总数            |
|cat_count |number       |分类总数             |
|created_at|string       |ISO 时间戳          |
|updated_at|string       |ISO 时间戳          |

**对象存储：quotation_rows**（报价数据行）

|字段            |类型           |说明                                  |
|--------------|-------------|------------------------------------|
|id            |自增整数（keyPath）|主键                                  |
|file_id       |number（索引）   |关联 quotation_files.id               |
|file_name     |string       |冗余，方便展示                             |
|category      |string       |所属分类名，如「HP系列」                       |
|category_order|number       |分类排序                                |
|model         |string（索引）   |型号值，如「CC388A」                       |
|header_data   |Array        |表头列名，如 [“序号”,“型号”,“适用机型”,“单价”]      |
|row_data      |Array        |数据值，如 [“1”,“CC388A”,“HP M1136”,“85”]|
|row_order     |number       |行排序                                 |
|created_at    |string       |ISO 时间戳                             |

### 5.3 db.js 函数对照表

`js/db.js` 封装所有 IndexedDB 操作，供 search.js 和 admin.js 调用：

|函数                                   |功能                              |
|-------------------------------------|--------------------------------|
|`db.init()`                          |初始化数据库，创建对象存储和索引                |
|`db.search(q, fileId?)`              |模糊搜索所有行（遍历匹配 model + row_data）  |
|`db.getFiles()`                      |获取所有文件列表                        |
|`db.getFile(id)`                     |获取单文件详情（含按分类分组的数据）              |
|`db.saveFile(fileRecord, rows)`      |保存文件元信息及所有数据行                   |
|`db.deleteFile(id)`                  |删除文件及所有关联数据行                    |
|`db.updateFile(id, fileRecord, rows)`|替换文件（先删除旧数据，再写入新数据）             |
|`db.exportFile(id)`                  |读取文件所有行数据，供导出 Excel 使用          |
|`db.getStats()`                      |统计：文件总数 / 数据行总数 / IndexedDB 占用估算|

## 六、前端交互细节

### 6.1 结果渲染层级

```
.result-file                    ← 文件卡片容器
  .result-file-header           ← 蓝色标题栏（可折叠，▶旋转90°）
  .result-file-body
    .result-category            ← 分类容器
      .result-category-header   ← 灰色标题栏（缩进32px，可折叠）
      .result-category-body
        .table-wrapper          ← 横向滚动容器
          table.data-table      ← 数据表
            thead → th (sticky固定表头，浅蓝#e8f0fe)
            tbody → tr (斑马纹 nth-child(even) #fafbfc)
              td.highlight      ← 匹配单元格 (#fff3cd黄色)
              td.cell-truncated ← 截断单元格 (>30字，带🔍图标)
```

### 6.2 双击弹窗

- 触发：双击任意 `.cell-truncated` 格
- 显示：全屏遮罩 + 居中白色卡片
- 内容：列名标题 + 完整文本（pre-wrap）+ 关闭按钮
- 动画：fadeIn 背景 + popIn 卡片

### 6.3 文件筛选

- 搜索框下方显示所有文件的勾选框（药丸形标签）
- 默认全选，可单独取消/勾选
- 全选/取消全选按钮
- 勾选变化时自动重新搜索

### 6.4 管理后台

- 顶部统计卡片（文件数、总行数、存储量）
- 拖拽/点击上传区域（虚线边框，hover变色）
- 文件列表（每行：文件名、行数、分类数、时间 + 导出/更新/删除按钮）
- Toast 通知（右上角弹出）
